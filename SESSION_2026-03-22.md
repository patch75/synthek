# Session 2026-03-22 — Granulométrie + Refonte page Projet

## Résumé utilisateur

Nouvelle fonctionnalité opérationnelle : **import automatique de la granulométrie** depuis le fichier Excel architecte (plan masse / tableau de surface), avec restitution d'un tableau détaillé par bâtiment.

### Ce que vous pouvez faire maintenant

1. Ouvrir un projet → section **🏢 Bâtiments — Granulométrie**
2. Cliquer **📥 Importer Excel** → sélectionner le fichier architecte (.xlsx)
3. Vérifier la **proposition de regroupement** (tableau Montée → Bâtiment, modifiable)
4. Cliquer **✓ Confirmer et importer**
5. Le tableau synthèse s'affiche et est **persisté** — il réapparaît au rechargement

### Tableau synthèse D1

| Bâtiment | Montées | Logements | LLI | LLS | BRS | Acc.std | Acc.premium | Villas | Fiabilité |
|---|---|---|---|---|---|---|---|---|---|
| A | A1, A2 | 9 | 0 | 0 | 1 | 6 | 2 | 0 | haute |
| ... | | | | | | | | | |
| **TOTAL** | | **49** | | | | | | | |

### Règles métier appliquées automatiquement

- **PREMIUM** = logement au dernier niveau de sa montée, sans annotation sociale
- **VILLA** = logement dans un bloc nommé VILLA * (type produit distinct)
- **Financement** détecté via annotations (LLI) (LLS) (BRS) dans les N° de logement
- **Financement montée global** : ex. "C (LLS)" → tous logements sans annotation = LLS
- Surfaces m² ignorées, formules Excel non résolues ignorées

### Architecture pipeline granulométrie

```
Frontend (Projet.jsx)
  └─ POST /projets/:id/granulometrie/proposer  (fichier base64)
       └─ Backend (projets.js)
            └─ POST http://127.0.0.1:5001/granulometrie/proposer
                 └─ extraire_granulometrie(bytes, nom, regroupement_valide=None)
                      └─ Retourne : { montees_detectees, proposition_regroupement }

Frontend (Projet.jsx) — après validation utilisateur
  └─ POST /projets/:id/granulometrie/import  (fichier + regroupement)
       └─ Backend (projets.js)
            └─ POST http://127.0.0.1:5001/granulometrie/import
                 └─ extraire_granulometrie(bytes, nom, regroupement_valide={...})
                      └─ Retourne : JSON D1 complet
            └─ prisma.projet.update({ batimentsComposition: JSON.stringify(batiments) })
```

### Format JSON D1 stocké dans batimentsComposition

```json
[
  {
    "nom": "A",
    "montees": ["A1", "A2"],
    "nb_logements": 9,
    "LLI": 0, "LLS": 0, "BRS": 1,
    "acces_std": 6, "acces_premium": 2, "villas": 0,
    "fiabilite": "haute",
    "section_cctp": null,
    "feuilles_dpgf": []
  }
]
```

### Détection format D1 au rechargement (Projet.jsx)

```js
if (bats?.length && ('LLI' in bats[0] || 'acces_std' in bats[0])) {
  setGranulometreD1({ batiments: bats, total_logements: ..., ... })
  setImportGranuloStep(2)
}
```

### Routes parser Python (port 5001)

```
POST /granulometrie/proposer  { fichier: base64, nom_fichier }
  → { montees_detectees: [...], proposition_regroupement: { groupe: [montees] } }

POST /granulometrie/import    { fichier: base64, nom_fichier, regroupement: {} }
  → { batiments: [...], total_logements: N, donnees_manquantes: [...] }
```

### Validations Python

- **V1** : nb_logements = somme des typologies (tolérance zéro)
- **V5** : au moins un financement identifié par bâtiment non-villa

### Limites connues

- **PDF non supporté** — stub uniquement, lever NotImplementedError
- **Regroupement** : heuristique automatique, validation utilisateur obligatoire
- **section_cctp / feuilles_dpgf** : null après import — à renseigner manuellement dans la section Bâtiments ou via future UI de mapping CCTP/DPGF

## Problèmes connus non résolus

1. **LANCER.bat** : les fichiers `.bat` ne fonctionnent pas quand Claude Code tourne (port déjà occupé) — utiliser uniquement quand Claude Code est fermé
2. **Mapping CCTP/DPGF** : `section_cctp` et `feuilles_dpgf` restent null après import granulométrie — l'analyse Python diff reste désactivée tant que ce mapping n'est pas configuré
3. **Format ancien vs D1** : si un projet a des bâtiments en format ancien (nom + typologies), l'UI affiche la liste manuelle — pas de migration automatique
