## Diagnostics effectués

### Pourquoi Claude faisait de l'ingénierie de quantités
1. Faux C01 Python (`batiment=SECTION_X`) injectés → Claude voyait des "absences" et cherchait à les expliquer par les quantités
2. Texte DPGF brut envoyé → Claude peut lire les quantités directement
- Fix court terme : filtrage SECTION_X ✅
- Fix long terme : Claude reçoit uniquement les extraits d'écarts (Étape 4 PLAN_SYNTHESE_C.md)

### Pourquoi les alertes étaient limitées au BAT A
- Filtre `TEST TEMPORAIRE` + max 8 dans le prompt → Claude s'arrêtait à 8 alertes toutes dans le premier bâtiment analysé
- Fix : suppression des deux filtres ✅

### Cas "Évier vs Attente évier" (logements mixtes)
- Python génère C02/C03 sur des projets avec mix accession/BRS/social
- `Attente évier` (accession) ≠ `Évier` (social) : deux prestations distinctes à juste titre
- Python ne connaît pas le sous-périmètre par type de logement
- **Verdict : faux positif structurel** → marquer ✗ dans méthode légère, documenté dans FAUX_POSITIFS_PYTHON.md
